# Ponytail Debt Skill

## Purpose

Capture intentional shortcuts so temporary minimal solutions do not become permanent hidden debt.

## Debt Record Rules

- Every intentional ceiling must have a `ponytail:` note.
- Each note must define the upgrade trigger.
- Include owner and review window when possible.
- Keep debt scoped to specific files/behaviors.

## Priority Heuristics

1. Security or trust-boundary ceiling
2. Data-loss or recovery limitation
3. Accessibility limitation
4. Performance/scalability ceiling
5. Maintainability-only ceiling

## Output Format

- `Debt Item:` concise title.
- `Current Ceiling:` what was intentionally not implemented.
- `Upgrade Trigger:` explicit event/threshold/date.
- `Risk:` severity and affected surface.
- `Next Action:` minimal follow-up to retire debt.
